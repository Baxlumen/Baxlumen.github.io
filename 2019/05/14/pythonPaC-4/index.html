爬 虫——正则表达式(re)

Posted on 2019-05-14 | Edited on 2019-05-15

(正则之中最好不要随便加空格)

\d 匹配一个数字
\w 匹配一个数字或字母或下划线
. 匹配任意字符(一个/除了换行符)
、（跟md冲突了，淦）* 任意个字符(包括0个)
、（跟md冲突，淦）+ 至少一个

？ 0个或1个（加在后面）
\ (1)可以将一个普通字符变成一个特殊字符，例如\d表示匹配所有十进制数字
(2)接触元字符的特殊功能，例如.表示匹配 . 本身
(3)引用序号对应的子组(‘()’所括起来的)所匹配的字符串
(4)八进制转义符，即 \ 后加一个八进制数，然后根据ascii码来转义成相应字符

{n} n个(跟在要重复的元素后边)
{n,m} n到每个

[] 表示范围（只能匹配一个字符）
[^abc] 表示出了abc意外的都匹配
[0-9a-zA-Z_] 可以匹配数字字母下划线
（ \ 在[]里面是转义符，[]里面的内容全部为字符，比如[.]表示匹配一个.）
A|B A或B
^ 表示行的开头(或者 \A)
(当^在[]中且在最前面，表示除了[]之内的才匹配)
$ 表示行的结束(或者 \Z)
^\d 表示必须以数字开头
\d$ 表示必须以数字结尾
\b 匹配一个单词边界，单词定义为字母，数字，下划线
例如：
re.findall(r’\bfish\b’,’fish.com!fish_com!(fish)’)
[‘fish’, ‘fish’]
这里的fish_com会被认为是一个单词，所以不会被匹配，会匹配fish.com (fish)中的fish

\B 与\b 相反，匹配非单词边界，例如 py\B 会匹配 python py3 py2等，但不会匹配
py. py!等
\s 在str模式会匹配空白字符（\t \n \r \f \v ）

#1.re模块
pat 表示使用字符串表示的正则表达式
函数 描述
compile(pat,str) 根据正则表达式的字符串创建re对象
search(pat,str) 在字符串中查找pat
match(pat,str) 在指定字符串查找与pat匹配的子串，找得到返回Mathch object
split(pat,str) 根据pat来分割子串，返回一个list
findall(pat,str) 返回一个list，包含所有与pat匹配的子串
sub(pat,replcae,str) 从左到右将于pat匹配的子串替换为replace
escape(str) 对字符串中所有可能被视为正则运算符的字符进行转义

#2.匹配对象和编组（默认为0，即整个pat）
当查找与pat匹配的子串时，查找成功会返回MatchObject对象，编组是针对这种对象来进行的
方法 描述
group() 返回与pat中给定编组匹配的子串
start() 返回与给定编组匹配的子串的起始索引
end() 返回终止索引加1
span() 返回一个元组，包含与给定编组匹配的子串的起始索引和终止索引

#3.在sub()替换函数中使用编组
例如：

pat=r’*([^*]+)*‘
re.sub(pat,r’\1‘,’hello,world!’)
‘hello,world!’
在sub()函数中，replace参数位置，编组需要转义

注：：表达式 . 就是单个字符匹配任意次，即贪婪匹配。 表达式 .? 是满足条件的情况只匹配一次，即最小匹配.

#4.一些tips
search()方法用于在字符串中搜索正则表达式第一次出现的位置

第一个参数是正则，最好使用原始表达式r’’;其实是从0开始索引
消除通配符的功能，使用反斜杠 \

此时 . 不再是通配符，它只是一个 .（元字符）
当需要去匹配IP地址时，即要匹配0-255之间的数字
re.search(‘[0-255]’,’188’)
这句话表示，正则表达式是需要匹配首先一个0-2之间的数字，或者一个5，即只能匹配0,1,2,5这四个数字。匹配时都是字符串的格式，数字也是，对于字符串来说数字只有0-9这几个字符
所以要匹配0-255这个区间内的数字，应该
r’[01]\d\d|2[0-4]\d|25[0-5]’

#5.贪婪与非贪婪
贪婪是指尽可能多的去匹配，比如：

本意是匹配一个<>，但是由于贪婪匹配，会一直往下找到 > ，知道后面的没有了。即尽可能多的匹配

非贪婪，在表示重复的元字符后（+ * ）边加上 ?

#6.编译正则表达式
如果你需要重复的使用某个正则表达式，那么你可以先将正则表达式编译成模式对象，
我们用re.compile()方法来编译

import re
p = re.compile(r’[A-Z]’)
type(p)

p.search(‘I love me’)

<re.Match object; span=(0, 1), match=’I’>

p.findall(‘I LOVE FISH’)
[‘I’, ‘L’, ‘O’, ‘V’, ‘E’, ‘F’, ‘I’, ‘S’, ‘H’]

#7.编译标志

变易变质让你可以修改正则报答时的工作方法，编译表示均有两个名字：完整名和简写，另外，多个标志还可以同时使用(通过 | )，如re.I|re.M 就是同时设置I和M标志

#8.模块补充

re.search(pattern, string, flags=0)

可以看到，模式对象的参数后面有可选参数，是起始位置（rx.search(string, 0, 50) is equivalent to rx.search(string[:50], 0)）例如

并且search方法后返回的是一个匹配对象（ match object），可以使用一些方法(见#2)

##findall方法
如果正则中有子组(有被括号括起来的内容)，那么就会返回匹配到子组中的内容；如果有多个子组，就会返回一个tuple的形式
(If one or more groups are present in the pattern, return a list of groups; this will be a list of tuples if the pattern has more than one group)

那么当我们的正则中有需要使用括号的情况呢？例如，当获取网页上的代理IP时，正则为：
‘(([0,1]?\d?\d|2[0-4]\d|25[0-5]).){3}([0,1]?\d?\d|2[0-4]\d|25[0-5])’
此时使用findall()函数时，会返回一个元组，例如:

显然不是我们所期望的结果。解决方法，就是让子组不捕获内容，语法为(?:…),该子组匹配的字符串无法从后边获取。所以我们给上面的正则表达式进行处理：
‘(?:(?:[0,1]?\d?\d|2[0-4]\d|25[0-5]).){3}(?:[0,1]?\d?\d|2[0-4]\d|25[0-5])’
